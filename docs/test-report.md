## 问题记录（测试过程中追加）

### 问题 #1：登录页输入框 placeholder 显示异常

| 项目       | 内容                                                                                        |
|----------|-------------------------------------------------------------------------------------------|
| **模块**   | 登录页                                                                                       |
| **问题描述** | 输入框 placeholder 显示为"用户名: admin"和"密码: admin123"，看起来像是已填入的值，但实际上输入框为空，用户点击登录时会提示"请输入用户名/密码" |
| **严重程度** | 中                                                                                         |
| **复现条件** | 打开登录页面，不输入任何内容直接点击登录按钮                                                                    |
| **影响**   | 用户可能误解以为已填入默认值，造成困惑                                                                       |

### 问题 #2：知识库文档上传后自动 Embedding 未触发

| 项目       | 内容                                                                                |
|----------|-----------------------------------------------------------------------------------|
| **模块**   | 知识库管理                                                                             |
| **问题描述** | 使用非语义分块策略（如"按段落"）上传文档后，文档状态变为"已分块"，但不会自动触发 Embedding 计算，即使知识库已配置默认 Embedding 模型   |
| **严重程度** | 高                                                                                 |
| **复现条件** | 1. 创建知识库并设置默认 Embedding 模型 2. 上传文档，选择"按段落"分块策略 3. 等待分块完成 4. 状态为"已分块"而非自动进入"正在向量化" |
| **影响**   | 用户需要手动点击"向量化"按钮才能完成文档处理，增加操作步骤                                                    |

### 问题 #3：知识库详情页统计数据不刷新

| 项目       | 内容                                      |
|----------|-----------------------------------------|
| **模块**   | 知识库管理                                   |
| **问题描述** | 知识库详情页顶部的"文档数"和"分块数"始终显示为0，即使已上传文档并完成分块 |
| **严重程度** | 中                                       |
| **复现条件** | 进入知识库详情页，上传文档后观察顶部统计数据                  |
| **影响**   | 用户无法直观了解知识库的整体状态                        

### 问题 #4：数据集详情页数据项数显示不更新

| 项目       | 内容                                     |
|----------|----------------------------------------|
| **模块**   | 数据集管理                                  |
| **问题描述** | 数据集详情页顶部显示"数据项数 : 0"，即使已添加多条数据项，数量仍不刷新 |
| **严重程度** | 中                                      |
| **复现条件** | 进入数据集详情页，添加数据项后观察顶部的数据项数统计             |
| **影响**   | 用户无法直观了解数据集的数据量                        |

### 问题 #5：新建评估任务失败 (400 Bad Request)

| 项目       | 内容                                                      |
|----------|---------------------------------------------------------|
| **模块**   | 评估系统                                                    |
| **问题描述** | 创建评估任务时，填写名称、描述、选择数据集和对话模型后点击确定，后端返回400错误，弹窗不会关闭，任务创建失败 |
| **严重程度** | 高                                                       |
| **复现条件** | 1. 点击"新建评估任务" 2. 填写名称和描述 3. 选择已有数据集 4. 选择对话模型 5. 点击确定   |
| **影响**   | 用户无法创建新的评估任务                                            |

### RAG对话测试详情

**测试场景**: 电商平台智能客服FAQ

**第一轮对话**:

- 用户输入: "我想退货，已经收到商品5天了，还能申请退货吗？"
- AI回答: 正确引用知识库内容，说明7天无理由退货政策，并提示特殊商品例外

**第二轮对话**:

- 用户输入: "那退款多久能到账？"
- AI回答: 正确理解上下文，引用知识库说明退款1-3个工作日到账

**结论**: RAG多轮对话功能正常，能正确检索知识库内容并理解对话上下文。

### 修复详情

#### 问题 #1: 登录placeholder显示异常

- **修复**: 将placeholder从"用户名: admin"改为"请输入用户名"
- **保留**: 底部提示信息"默认账号: admin / admin123"仍显示

#### 问题 #2: 知识库自动Embedding未触发

- **修复**: 放宽触发条件，不再要求模型配置已有dimension和tableName
- **增强**: embedChunksAsync方法现在能动态获取dimension并创建向量表

#### 问题 #3: 知识库统计数据不刷新

- **修复**: 使用正确的字段chunksCreated而非totalChunks
- **增强**: 在文档处理完成后自动调用knowledgeBaseService.updateStatistics()

#### 问题 #4: 数据集数据项数不刷新

- **修复**: 添加/删除数据项后同时刷新数据集列表
- **增强**: fetchDatasets方法自动更新selectedDataset状态

#### 问题 #5: 新建评估任务400错误

- **修复**: promptTemplateId变为可选字段
- **增强**: 未提供模板时使用默认提示词
