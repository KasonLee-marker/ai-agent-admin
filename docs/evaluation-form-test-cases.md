# 评估表单联动测试用例

## 测试环境准备

### 前置条件

- 后端服务运行正常 (http://localhost:8080)
- 前端服务运行正常 (http://localhost:5173)
- 已存在测试数据：
    - 至少 1 个数据集（有期望输出）
    - 至少 1 个数据集（无期望输出）
    - 至少 1 个 Prompt 模板
    - 至少 1 个对话模型（CHAT 类型）
    - 至少 1 个 Embedding 模型（EMBEDDING 类型）
    - 至少 1 个知识库

### TC-001: 基础字段必填验证

| 项目       | 内容                        |
|----------|---------------------------|
| **测试目的** | 验证基础必填字段为空时的表单提交拦截        |

## 自动化测试脚本 (Playwright MCP)

以下测试用例可通过 Playwright MCP 自动化执行：

### 自动化测试脚本 - TC-003

```javascript
// TC-003: enableRag 开关开启 - 知识库未选择
async (page) => {
  // 导航到评估页面
  await page.goto('http://localhost:5173/evaluations');
  
  // 点击新建按钮
  await page.getByRole('button', { name: '新建评估任务' }).click();
  
  // 填写名称
  await page.getByLabel('名称').fill('测试任务');
  
  // 选择数据集
  await page.getByLabel('数据集').click();
  await page.getByRole('option').first().click();
  
  // 开启 RAG 开关
  await page.getByRole('switch', { name: '启用RAG评估' }).click();
  
  // 检查知识库字段变为必填
  const kbLabel = page.locator('label:contains("知识库")');
  await expect(kbLabel).toHaveText(/知识库.*\*/);
  
  // 检查提示信息
  await expect(page.locator('text=RAG模式需要选择知识库')).toBeVisible();
  
  // 提交并检查错误
  await page.getByRole('button', { name: '确定' }).click();
  await expect(page.locator('text=启用RAG评估时必须选择知识库')).toBeVisible();
}
```
